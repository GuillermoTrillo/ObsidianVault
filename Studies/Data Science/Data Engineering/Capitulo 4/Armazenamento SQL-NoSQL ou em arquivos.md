**Depende dos padrões de acesso.**


**SQL:** formato tradicional de armazenamento no formato tabular. É um dos mais antigos e dos mais utilizados. Ideal para dados estruturados. Usamos SGBDs (Sistemas Gerenciadores de Banco de Dados) como Oracle, PostgreSQL, SQL Server ou MySQL, entre outros.

**NoSQL:** O armazenamento NoSQL nasceu na era do Big Data para permitir o armazenamento de dados em diferentes formatos, em especial, dados semi-estruturados no formato JSON, XML ou colunar. São orientados à performance e facilidade de uso. Exemplos de SGBDs NoSQL: MongoDB, Redis, Apache Cassandra, Apache HBase e Amazon DynamoDB.

# Big Data
**Big Data** é definido por 4 Vs:

**V**olume, **V**elocidade, **V**ariedade e **V**eracidade.

V de variedade é normalmente uma das partes mais complexas. Pode ser necessário trabalhar com dados no formato de vídeo, áudio, imagens, texto, arquivos pdf ou mesmo ícones. Ou ainda dados em um formato que facilite a pesquisa e reduza o espaço necessário para armazenamento. Bancos de dados SQL ou NoSQL podem não ser ideais nesses casos, isso sem falar na performance de acesso. É nesses casos onde precisaremos de alternativas, de formatos e diferentes sistemas. Alternativas incluem:
Parquet, Avro, JSON, CVS, ORC, HDF5 e vários outros formatos.
